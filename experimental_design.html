Introduction

This section introduces the experimental design for creating an early-earthquake detection system using machine learning. The design involves hardware components such as piezoelectric transducers and microcontrollers, as well as software algorithms for data analysis.

Materials and Methods

The system uses Zirconate Titanate (PZT) piezoelectric discs to convert mechanical strain into electrical signals. These signals are conditioned and amplified, then processed by an ESP32 microcontroller attached to a sensor node. Power management relies on rechargeable Li-ion batteries and wake-on-radio communication to minimize energy consumption.

Experimental Setup

The system is tested on a controlled vibration setup using a shake table, which simulates earthquake vibrations within the frequency range of 0.5-20Hz. Data from sensors is transmitted wirelessly via LoRa modules to a Raspberry Pi, where machine learning algorithms analyze the signals.

Data Collection & Analysis

Data includes voltage readings, vibration metrics, and system latency. These are processed to evaluate detection accuracy, false positives, energy efficiency, and response times. Charts and tables will be generated to compare the system’s performance against traditional detection methods.

Expected Outcomes

The goal is an affordable, energy-efficient early warning system that can be deployed in resource-limited regions with minimal maintenance requirements.
